## 一、 后端技术需求设计 (Backend Design)

后端作为纯粹的 API 服务（API-First），主要负责多源文档解析、切分、向量化入库以及高精度的 RAG 混合检索服务。

### 1. 基础架构与运行环境

- **开发语言与框架**：Python 3.10.14 + FastAPI。
- **环境隔离**：依托本地 `conda` 中的环境进行统一管理，你需要创建新的conda环境，并确保 LlamaIndex、Qdrant-Client 等依赖版本的绝对一致性。
- **并发与异步**：FastAPI 需全量开启 `async/await` 异步处理，尤其是在调用云端 Embedding 和Reranker 模型时，避免阻塞主线程。

### 2. 核心 API 接口定义

#### 2.1 数据治理与入库模块 (Ingestion API)

- **`POST /api/v1/kb/create`**
  - **功能**：创建一个新的知识集。
  - **参数**：`knowledge_base_id` (知识集唯一标识，随机生成可以按照一定规则生成), `description`。
  - **逻辑**：在底层的 Qdrant 中动态初始化一个同名的 Collection，并配置 Sparse Vector 和 Dense Vector 的双路索引参数。
- **`POST /api/v1/document/upload`**
  - **功能**：多格式文档上传与预处理，支持对知识集增量添加和定点删除。
  - **参数**：`file` (FormData), `knowledge_base_id`。
  - **逻辑**：接收 PDF/Word/Excel 等文件，调用前置版面解析工具转为标准 Markdown。生成全局唯一 `doc_id` 并注入 Metadata，随后进入异步的 Chunking（切分）与 Embedding（向量化）流水线。

#### 2.2 核心检索服务模块 (Retrieval API)

- **`POST /api/v1/retrieve`** (⭐ 核心基座接口)

  - **功能**：接收外部系统（如 WeLink 部门智能体等下游应用）的查询请求，返回高精度的上下文片段。

  - **参数**：

    JSON

    ```
    {
      "user_id": "admin_01",
      "knowledge_base_id": "tech_support_kb",
      "query": "如何处理网关连接失败报错？",
      "top_k": 10,
      "top_n": 3
    }
    ```
