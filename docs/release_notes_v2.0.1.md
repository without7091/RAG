# RAG v2.0.1 发布说明

> 发布日期：2026-03-08
> 适用范围：知识库管理、数据治理、检索链路

## 概述

`v2.0.1` 主要聚焦三个方向：

1. 增强知识集目录管理能力，支持项目 / 子目录 / 知识集的树形组织；
2. 升级检索链路，补充查询改写、调试信息与流式状态反馈；
3. 优化数据治理页交互体验，修复创建后选中、目录展开状态、文件勾选残留等问题。

本次版本适合希望进一步提升知识库组织能力、检索可观测性和后台运营体验的场景。

## 重点更新

### 1. 知识集目录树能力上线

- 新增知识集目录模型、Schema 与服务层，支持目录化管理知识集；
- 支持一级目录 / 二级目录的创建、编辑、删除；
- 知识集列表升级为树形结构展示，便于按项目或业务域组织知识资产；
- 新增目录树接口与相关前端数据结构，前后端能力保持一致。

### 2. 数据治理体验优化

- 新建项目后，左侧目录自动切换到新创建的项目；
- 新建知识集后，自动选中新创建的知识集，不再停留在旧选择上；
- 目录展开 / 收起状态支持保留，不再因刷新、创建、切换而全部展开；
- 切换知识集时自动清空文件勾选，避免跨知识集误操作；
- 目录选中与知识集选中解耦，右侧文档区空态与当前选择保持一致。

### 3. 检索链路增强

- 检索接口支持可选的查询改写能力；
- 支持基于查询改写的 fanout 检索与结果聚合；
- 检索响应补充调试信息，便于分析检索命中过程；
- 检索流式返回增加状态事件，便于前端展示改写 / 检索执行阶段；
- 检索相关异常处理、依赖注入和服务封装进一步完善。

## 后端改动

### 知识库与目录管理

- 新增目录相关数据模型与服务；
- 扩展知识库接口以支持目录树查询与目录归属；
- 更新知识库相关 Schema 与 API 文档；
- 补充目录与知识库联动的单元测试、集成测试。

### 检索与查询改写

- 新增查询改写服务；
- 新增 OpenAI 兼容的 Chat Completion 调用封装，用于改写场景；
- 扩展检索请求 / 响应结构，支持调试信息与流式处理；
- 优化检索服务内部流程与错误处理逻辑。

## 前端改动

### 数据治理页

- 新增目录树组件与目录操作弹窗；
- 左侧树支持目录选中、知识集选中、展开收起状态持久化；
- 创建知识集 / 目录后支持精确定位到最新创建对象；
- 文档勾选状态随知识集切换自动重置。

### Playground 与检索体验

- 对接新的检索接口能力；
- 支持更细粒度的状态展示与交互反馈；
- 相关查询输入和 SSE 进度展示已同步适配。

## 配置与接口变更

- 新增查询改写相关配置项；
- 检索接口请求 / 响应结构有扩展；
- 知识库接口新增目录树与目录管理相关能力；
- 如有自定义前端调用或外部集成，请同步检查 `backend/API.md` 与 `frontend/src/lib/api.ts`。

## 测试与验证

本次版本已补充并覆盖以下方向：

- 知识库目录服务测试；
- 查询改写服务测试；
- 检索路由与检索服务测试；
- 知识库 API 集成测试；
- 数据治理页前端交互在构建与 lint 层面完成验证。

## 升级提示

- 若当前环境基于旧版知识库平铺结构，请关注目录相关接口与数据结构变化；
- 若前端有自定义调用检索接口，请同步适配新增字段；
- 若部署启用了查询改写，请确认对应模型地址、鉴权信息与相关配置已正确填写。
