## 2. 后端 Dockerfile

多阶段构建，基于 `python:3.10-slim`，包含 FastEmbed 本地模型缓存支持。

```dockerfile
# ============================================================
# Stage 1: Builder — 安装依赖
# ============================================================
FROM python:3.10-slim AS builder

WORKDIR /build

# 系统依赖（编译 C 扩展需要）
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc g++ libffi-dev && \
    rm -rf /var/lib/apt/lists/*

COPY pyproject.toml ./

# 安装 Python 依赖到 /install 前缀
RUN pip install --no-cache-dir --prefix=/install ".[local-sparse]"

# ============================================================
# Stage 2: Runtime — 最小化镜像
# ============================================================
FROM python:3.10-slim AS runtime

WORKDIR /app

# 复制安装好的 Python 包
COPY --from=builder /install /usr/local

# 复制应用代码
COPY app/ ./app/

# 创建数据目录
RUN mkdir -p /app/data/uploads \
             /app/data/qdrant_storage \
             /app/data/fastembed_cache

# 环境变量默认值
ENV PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    HF_HUB_OFFLINE=1 \
    LLAMA_INDEX_DISABLE_TELEMETRY=true \
    UPLOAD_DIR=/app/data/uploads \
    QDRANT_STORAGE_PATH=/app/data/qdrant_storage \
    FASTEMBED_CACHE_PATH=/app/data/fastembed_cache \
    SQLITE_DATABASE_URL=sqlite+aiosqlite:///./data/metadata.db

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "1"]
```

**关键设计决策：**

- `HF_HUB_OFFLINE=1`：禁止容器运行时联网下载 HuggingFace 模型，模型文件通过 Volume 或构建时预置
- `--workers 1`：由于使用 asyncio 事件循环和内存中的 Qdrant 本地存储，单 Worker 是必须的
- 数据目录通过 Volume 挂载到宿主机

## 3. 前端 Dockerfile

基于 `node:20-alpine` 的 Next.js standalone 多阶段构建。

```dockerfile
# ============================================================
# Stage 1: Dependencies
# ============================================================
FROM node:20-alpine AS deps

WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci --omit=dev

# ============================================================
# Stage 2: Builder
# ============================================================
FROM node:20-alpine AS builder

WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

# 构建时设置后端 API 地址（Next.js rewrites 在运行时生效）
ENV NEXT_TELEMETRY_DISABLED=1

RUN npm run build

# ============================================================
# Stage 3: Runner — standalone 模式
# ============================================================
FROM node:20-alpine AS runner

WORKDIR /app

ENV NODE_ENV=production \
    NEXT_TELEMETRY_DISABLED=1

# 创建非 root 用户
RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 nextjs

# 复制 standalone 输出
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

USER nextjs

EXPOSE 3000

ENV PORT=3000 \
    HOSTNAME="0.0.0.0"

CMD ["node", "server.js"]
```

> **注意：** 需要在 `next.config.ts` 中确认 `output: "standalone"` 配置已启用。当前配置中已通过 `proxyClientMaxBodySize: "200mb"` 支持大文件上传，API 代理规则将 `/api/:path*` 转发到后端。

## 4. Docker Compose 编排

```yaml
version: "3.8"

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: rag-backend
    restart: unless-stopped
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data          # 持久化: Qdrant存储、SQLite、上传文件、模型缓存
    env_file:
      - .env
    environment:
      - UPLOAD_DIR=/app/data/uploads
      - QDRANT_STORAGE_PATH=/app/data/qdrant_storage
      - FASTEMBED_CACHE_PATH=/app/data/fastembed_cache
      - SQLITE_DATABASE_URL=sqlite+aiosqlite:///./data/metadata.db
      - HF_HUB_OFFLINE=1
    healthcheck:
      test: ["CMD", "python", "-c", "import httpx; httpx.get('http://localhost:8000/docs')"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 30s

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: rag-frontend
    restart: unless-stopped
    ports:
      - "3000:3000"
    environment:
      # Next.js API rewrite 目标地址
      - BACKEND_URL=http://backend:8000
    depends_on:
      backend:
        condition: service_healthy

volumes:
  rag-data:
```

**Volume 结构说明：**

```
./data/
├── uploads/              # 用户上传的原始文件
│   └── kb_{id}/          # 按知识库隔离
│       └── document.pdf
├── qdrant_storage/       # Qdrant 向量数据（本地磁盘模式）
├── fastembed_cache/      # FastEmbed SPLADE 模型文件
│   └── Qdrant--bm25/    # 具体模型缓存
└── metadata.db           # SQLite 元数据库
```

## 6. FastEmbed 模型离线缓存

内网部署时无法联网下载模型，需要提前在有网环境准备缓存。

### 步骤 1：在有网环境下载模型

```bash
# 创建临时 Python 环境
pip install fastembed

# 下载 SPLADE 模型到指定目录
python -c "
from fastembed import SparseTextEmbedding
import os
os.environ['FASTEMBED_CACHE_PATH'] = './fastembed_cache'
model = SparseTextEmbedding(model_name='Qdrant/bm25')
# 触发下载
list(model.embed(['test']))
print('Model downloaded successfully')
"
```

### 步骤 2：打包模型缓存

```bash
tar -czf fastembed_cache.tar.gz fastembed_cache/
```

### 步骤 3：部署到内网

```bash
# 在内网服务器解压到 data 目录
tar -xzf fastembed_cache.tar.gz -C ./data/

# 确认目录结构
ls ./data/fastembed_cache/
# 应包含 Qdrant--bm25/ 子目录
```

### 步骤 4：配置离线模式

```env
# .env 中确保以下配置
SPARSE_EMBEDDING_MODE=local
FASTEMBED_CACHE_PATH=./data/fastembed_cache
FASTEMBED_MODEL_NAME=Qdrant/bm25
```

容器中设置 `HF_HUB_OFFLINE=1` 确保不会尝试联网下载。

## 7. 内网部署配置示例

当公司内网部署了自建的 Embedding/Reranker 推理服务时，修改 `.env`：

```env
# ─── 内网推理服务地址 ───
EMBEDDING_URL=http://192.168.1.100:8080/v1/embeddings
RERANKER_URL=http://192.168.1.100:8080/v1/rerank

# 如果内网也部署了 Sparse Embedding API 服务
SPARSE_EMBEDDING_MODE=api
SPARSE_EMBEDDING_URL=http://192.168.1.100:8080/v1/sparse-embeddings
SPARSE_EMBEDDING_MODEL=Qdrant/bm25

# 如果内网使用不同的模型名
EMBEDDING_MODEL=Qwen/Qwen3-Embedding-4B
RERANKER_MODEL=Qwen/Qwen3-Reranker-4B
```

**内网推理服务要求：**

- Embedding API 需兼容 OpenAI `/v1/embeddings` 格式
- Reranker API 需兼容 `/v1/rerank` 格式（Jina/Cohere 兼容）
- 网络延迟建议 < 50ms（同一数据中心内）

## 8. 服务注册接入方案

### Nacos 接入框架

```python
# 以下为接入 Nacos 的框架代码，需要在 main.py 的 lifespan 中集成

import httpx

NACOS_SERVER = "http://nacos-server:8848"
SERVICE_NAME = "rag-middleware"
SERVICE_IP = "192.168.1.50"   # 本机 IP
SERVICE_PORT = 8000

async def register_to_nacos():
    """注册服务到 Nacos"""
    async with httpx.AsyncClient() as client:
        await client.post(
            f"{NACOS_SERVER}/nacos/v1/ns/instance",
            params={
                "serviceName": SERVICE_NAME,
                "ip": SERVICE_IP,
                "port": SERVICE_PORT,
                "healthy": "true",
                "metadata": '{"version":"0.1.0","type":"rag-middleware"}',
            },
        )

async def deregister_from_nacos():
    """从 Nacos 注销服务"""
    async with httpx.AsyncClient() as client:
        await client.delete(
            f"{NACOS_SERVER}/nacos/v1/ns/instance",
            params={
                "serviceName": SERVICE_NAME,
                "ip": SERVICE_IP,
                "port": SERVICE_PORT,
            },
        )
```

### Eureka 接入框架

```python
# Eureka 注册框架
EUREKA_SERVER = "http://eureka-server:8761"
APP_NAME = "RAG-MIDDLEWARE"

async def register_to_eureka():
    async with httpx.AsyncClient() as client:
        await client.post(
            f"{EUREKA_SERVER}/eureka/apps/{APP_NAME}",
            json={
                "instance": {
                    "hostName": SERVICE_IP,
                    "app": APP_NAME,
                    "ipAddr": SERVICE_IP,
                    "port": {"$": SERVICE_PORT, "@enabled": "true"},
                    "status": "UP",
                    "dataCenterInfo": {
                        "@class": "com.netflix.appinfo.InstanceInfo$DefaultDataCenterInfo",
                        "name": "MyOwn",
                    },
                }
            },
            headers={"Content-Type": "application/json"},
        )
```

> **集成位置：** 在 `app/main.py` 的 `lifespan` 上下文管理器中，`yield` 前调用注册，`yield` 后调用注销。

## 9. VMware 预测试工作流

在正式部署到物理服务器前，建议先在 VMware 虚拟机中验证。

### 测试清单

```
□ 1. 准备 VM（Ubuntu 22.04, 8GB RAM, 4 CPU）
□ 2. 安装 Docker + Docker Compose
□ 3. 传输项目文件和模型缓存到 VM
□ 4. 解压 FastEmbed 模型缓存到 ./data/fastembed_cache
□ 5. 配置 .env（API Key、内网地址）
□ 6. docker compose build（首次构建）
□ 7. docker compose up -d
□ 8. 验证后端健康：curl http://localhost:8000/docs
□ 9. 验证前端加载：浏览器访问 http://<vm-ip>:3000
□ 10. 功能测试：
    □ 创建知识库
    □ 上传文档（PDF、DOCX、TXT）
    □ 触发向量化，观察状态流转
    □ 执行检索查询，验证结果
□ 11. 性能观察：
    □ 文档处理速度
    □ 检索响应延迟
    □ 内存占用（docker stats）
□ 12. 日志检查：docker compose logs backend
```

### 快速验证脚本

```bash
#!/bin/bash
# verify.sh — 快速验证部署状态
set -e

API="http://localhost:8000/api/v1"

echo "=== 1. 健康检查 ==="
curl -s "$API/../docs" | head -1 && echo " [OK]"

echo "=== 2. 创建知识库 ==="
KB_RESP=$(curl -s -X POST "$API/kb/create" \
  -H "Content-Type: application/json" \
  -d '{"knowledge_base_name":"test-kb","description":"deployment test"}')
echo "$KB_RESP"
KB_ID=$(echo "$KB_RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['knowledge_base_id'])")
echo "KB_ID=$KB_ID"

echo "=== 3. 上传文档 ==="
echo "This is a test document for deployment verification." > /tmp/test.txt
curl -s -X POST "$API/document/upload?knowledge_base_id=$KB_ID" \
  -F "file=@/tmp/test.txt"

echo ""
echo "=== 4. 列出文档 ==="
curl -s "$API/document/list/$KB_ID" | python3 -m json.tool

echo "=== 部署验证完成 ==="
```

## 10. 网络问题排查清单

| # | 问题现象 | 可能原因 | 排查方法 |
|---|---------|---------|---------|
| 1 | Embedding API 超时 | 内网 DNS 解析失败或代理阻断 | `curl -v $EMBEDDING_URL`，检查连通性 |
| 2 | Reranker 返回 401 | API Key 配置错误或过期 | 检查 `.env` 中 `SILICONFLOW_API_KEY` |
| 3 | FastEmbed 模型加载失败 | 缓存目录为空或权限不足 | `ls -la ./data/fastembed_cache/`，确认 `HF_HUB_OFFLINE=1` |
| 4 | 前端无法连接后端 | Docker 网络隔离或代理配置错误 | `docker exec rag-frontend wget -O- http://backend:8000/docs` |
| 5 | Qdrant 写入失败 | 磁盘空间不足或目录权限 | `df -h`，`ls -la ./data/qdrant_storage/` |
| 6 | 文档上传失败（413） | Nginx/代理层 body size 限制 | 检查反向代理的 `client_max_body_size` 配置 |

### 日志级别调整

```env
# 调试模式下增加日志详情
LOG_LEVEL=DEBUG
```

后端日志格式为：`%(asctime)s [%(levelname)s] %(name)s: %(message)s`

关键日志来源：
- `app.services.pipeline_worker` — Pipeline 调度与执行
- `app.services.embedding_service` — Embedding API 调用
- `app.services.vector_store_service` — Qdrant 操作

## 11. K8s 部署参考

### Backend Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: rag-backend
  labels:
    app: rag-backend
spec:
  replicas: 1    # 单副本：Qdrant 本地存储 + SQLite 不支持多副本
  selector:
    matchLabels:
      app: rag-backend
  template:
    metadata:
      labels:
        app: rag-backend
    spec:
      containers:
        - name: backend
          image: rag-backend:latest
          ports:
            - containerPort: 8000
          env:
            - name: HF_HUB_OFFLINE
              value: "1"
            - name: UPLOAD_DIR
              value: /app/data/uploads
            - name: QDRANT_STORAGE_PATH
              value: /app/data/qdrant_storage
            - name: FASTEMBED_CACHE_PATH
              value: /app/data/fastembed_cache
          envFrom:
            - secretRef:
                name: rag-secrets    # 包含 SILICONFLOW_API_KEY 等敏感配置
          volumeMounts:
            - name: rag-data
              mountPath: /app/data
          resources:
            requests:
              cpu: "2"
              memory: 4Gi
            limits:
              cpu: "4"
              memory: 8Gi
          readinessProbe:
            httpGet:
              path: /docs
              port: 8000
            initialDelaySeconds: 15
            periodSeconds: 10
          livenessProbe:
            httpGet:
              path: /docs
              port: 8000
            initialDelaySeconds: 30
            periodSeconds: 30
      volumes:
        - name: rag-data
          persistentVolumeClaim:
            claimName: rag-data-pvc
```
